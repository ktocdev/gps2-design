Notes

  • --color-bg-canvas: #000000 is reserved for the 3D/WebGL viewport background

Notes

  • Body font is Nunito (replaced Inter) — loaded via colors_and_type.css at wght 300–700 + italic
  • Base font size is 16px on html; all rem values scale from this root
  • Responsive heading scale: h1–h3 scale ×1.125 at 640px, ×1.25 at 1024px via calc() inside media queries
  • --font-weight-heading is an alias for 700 — always use for h1–h6 elements
  • -webkit-font-smoothing: antialiased applied globally on body
  • Heading font (Gaegu) is a handwritten-style display face — avoid using below 14px
  • Georgia and Caveat are not global tokens — scoped exclusively to AdoptionCertificate.jsx; do not use elsewhere
  • --font-family-stats (Roboto) is used only in stats.css stat grid components

Elevation Scale

Notes

  • All shadows use rgb(0 0 0 / alpha) modern syntax — not compatible with IE11
  • Panel resting state: --shadow-sm; hover state: --shadow-md (set via transition: all var(--transition-normal))
  • Modal / dialog elevation: --shadow-xl
  • Disabled buttons: box-shadow: none !important overrides all elevation
  • Colored hover shadows (primary/danger/warning) are per-component RGBA values at 0.1 opacity — not tokens
  • Tooltip backdrop blur: backdrop-filter: blur(1px) used on loading panel overlay, not as a token

Notes

  • All layers increment by 10, leaving room for intermediate values if a future component needs them
  • Tooltip (1070) is always the ceiling — it must appear above modals and popovers per UX convention
  • 3D WebGL canvas (--color-bg-canvas: #000000) lives at z-index: 0; all UI layers stack above
  • FAB subnavigation uses position: fixed — respect --z-index-fixed (1030)
  • Do not use arbitrary z-index values in components — always reference a token

Transitions

3 speeds · ease-in-out easing
TokenValueUsageDemo ↗ hover
--transition-fast 150ms ease-in-out Button states, focus rings, hover color shifts, tooltip opacity fade
--transition-normal 250ms ease-in-out Panel hover states, accordion expand/collapse, page-level transitions
--transition-slow 350ms ease-in-out Modal open/close, slide-in drawers, complex layout animations

Notes

  • @media (prefers-reduced-motion: reduce) — transitions set to none in panel, button, and tooltip components
  • Button hover: adds translateY(-1px); active state removes it — both via --transition-fast
  • Avoid transition: all on performance-critical components; specify individual properties explicitly
  • 3D WebGL animations and frame pacing are managed separately outside this token system
  • Tooltip opacity fade uses --transition-fast; the tooltip itself has no transform animation

Notes

  • All textures are pure CSS — no image files, no external assets; safe for offline use. Cage floor and 3D backdrop surfaces are handled by the 3D app.
  • Three wood variants exist: SimTopBar plank (#c2700d–#87470b), cage frame / shelf (#b45309–#78350f), and bonded pair slot (#d97706–#b45309) — they share the same grain technique but differ in base color and hairline spacing
    • used across SimTopBar, Tutorial, HelpOverlay, cage popovers, and banners
  • Awning stripe always sits at the top of its panel, immediately above the parchment body; paper grain overlays the parchment body below the stripe
  • Brass studs (radial-gradient(circle at 30% 30%, #fde68a, #1c1917)) are always 4-corner positioned, aria-hidden="true"
  • Nailed sign labels always carry a slight rotation (±1–2°) to feel hand-placed; never exceed ±3°
  • Wood grain overlay (rgba(255,255,255,.04) repeating) must sit in a position:absolute; inset:0; pointer-events:none div — do not apply directly to the background shorthand
  • The Ornate Parchment pattern is exclusive to AdoptionCertificate.jsx — use Simple Parchment everywhere else
  • Awning stripe height varies: 10px (compact banners), 12px (item popovers), 14px (SimTopBar dropdowns), 18px (HelpOverlay full-screen)
