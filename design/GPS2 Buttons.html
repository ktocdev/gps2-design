Behavior Notes

Non-visual · animation · truncation · conditional visibility

Primary Button

  • Press animation uses :active, not click. The translateY(+2px) + shadow-reduce lands in 100ms.
  • ::before top-highlight spans the full pill including any lead icon — ensuring uniform sheen across all variants.
  • Header tab (active) uses Gaegu 700 19px; store/modal Add/Adopt uses Nunito 700 13–15px. Same background, different font family.
  • Never truncate the label. Buttons are always single-line; expand width rather than clip or ellipsis.
  • Use the disabled attribute directly — do not fake it with only a CSS class. The attribute also removes it from the tab order.

Sign Button (SimTopBar)

  • Active/open state lifts upward (translateY(−2px)) rather than pressing down — signals a panel is open.
  • Badge is conditionally rendered: hidden when badge === 0. Never show a "0" badge.
  • Pig health dot: green (#178740) when all needs satisfied; inline badge (#dc2626 / #ea580c) with worst-need icon + count for warning/critical.
  • Each pill carries a per-position rotate(±0.5–1deg) tilt to look hand-arranged. Do not exceed ±2deg.
  • Click outside any open panel closes all via mousedown on document. The tutorial overlay is explicitly excluded from this handler.

Pause Button — Dual State

  • Same element toggles between green (playing) and amber (paused) states on click. Color and label transition simultaneously.
  • Label slots: "⏸ Pause" (playing) ↔ "▶ Resume" (paused).
  • The entire right-zone (coins + ? + pause) is hidden before pigs are adopted — pigsAdopted === 0.

Dashed Button

  • Used only alongside a Primary CTA when a secondary escape-hatch is needed (e.g. "Choose different pigs" next to "Adopt").
  • On hover, fills with a very light cream tint (rgba(254,243,199,.5)) — not a solid fill. Keeps it visually subordinate.
  • Never use as the sole CTA on a surface. It must always be paired with a more prominent action.
